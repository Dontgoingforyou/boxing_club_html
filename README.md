# boxing_club_website

Статический сайт витрины; репозиторий подключён к **Vercel**.

## Продакшен и превью (вторая ссылка для коллеги)

| Что | Ветка | Ссылка |
|-----|--------|--------|
| Публичный сайт для всех | `main` | прод-домен проекта (например `…-one.vercel.app` и ваши домены) |
| Версия «только отправили коллеге» | любая **не** `main`, например `preview` | **Preview Deployment** — отдельный `*.vercel.app` URL из сборки |

Как пользоваться:

1. Рабочие правки коммитите в ветку **`preview`** (или другую), не в `main`, пока не готовы к продакшену:
   - `git checkout preview` → правки → `git commit` → `git push origin preview`
2. После пуша откройте проект в [Vercel Dashboard](https://vercel.com/dashboard) → **Deployments** — у последнего деплоя с веткой `preview` будет своя ссылка **Visit**. Её и отправляйте коллеге.
3. Альтернатива: открыть **Pull Request** `preview` → `main` — в PR часто появляется ссылка на превью от Vercel.

Ветка **`preview`** создана как долгоживущая «песочница»: прод остаётся на `main`, превью обновляется при каждом пуше в `preview`.

**Важно:** превью-URL по сути не секретный — доступ ограничен только тем, что ссылку нигде не публикуют. Нужна именно защита паролем — см. в Vercel **Deployment Protection** (зависит от тарифа).
